DAPG: Distance-Aware Pruned Graph

Accurate and Efficient Approximate Nearest Neighbor Search

Solmaz Seyed Monir, Dr. Dongfang Zhao

University of Washington

View Code Project Page
+3.3% Recall 2.9× Faster Single Layer LSH Seeding

Abstract

DAPG introduces percentile-based local filtering and adaptive global sparsification to build degree-adaptive proximity graphs that preserve reachability while reducing redundant edges.

DAPG improves latency-recall trade-offs over state-of-the-art ANN baselines without multi-layer indexing.

Why This Work

Page Visits
Common gaps in existing ANN methods How DAPG addresses these
Fixed-degree graphs: Static degree limits prevent adaptive sparsification. Adaptive sparsity: Percentile-based filtering and global capping balance recall and cost.
Costly rebuilds: Traditional structures require full index reconstruction. Incremental updates: Supports amortized insertion with bounded rewire cost.
Uncontrolled expansion: Greedy traversal expands redundant nodes. Pruned search: Expansion is capped by β(ℓ), improving efficiency.
Weak theoretical link: Prior heuristics lack formal complexity bounds. Theory-backed: DAPG gives sublinear query complexity and bounded connectivity.
DAPG achieves higher recall, lower query latency, and reduced memory footprint while preserving theoretical query complexity.

Contributions

Contribution Description
1. Theory Formalizes distance-aware pruning and adaptive degree control in proximity graphs.
2. Method Introduces local percentile filtering and global capping for degree-adaptive graphs.
3. Empirics Outperforms LSH-APG across DEEP1M, MNIST, and SIFT1M.
4. Guidance Provides design rules for local and global pruning rates under dynamic insertions.

What DAPG Adds

  • Distance-aware local pruning using percentile thresholds per node.
  • Adaptive global sparsification to cap high-degree tails.
  • Parallel construction with thread-safe updates.
  • Serialization of graph and LSH structures for fast reloads.
  • Dynamic insertion and deletion with degree-stable connectivity.

Cost Model

C_Q = O(d̄_DAPG · β(ℓ))
T_Q = O(d · d̄_DAPG · β(ℓ))
β(ℓ) = O(log n) under small-world routing

The expected query cost factorizes into average degree and expansion depth.

Compilation

Linux

Requires C++11 and OpenMP.

cd ./cppCode/DAPG
make

Windows

Use Visual Studio 2019+ and import the project located in: 

./cppCode/DAPG/src/

Enable OpenMP and C++11 support in the build settings.

Running DAPG

Command Format

./dapg datasetName

Example

cd ./cppCode/DAPG
./dapg sift

This runs DAPG index construction and search on the SIFT dataset.

Supported Datasets

  • Audio
  • SIFT1M
  • Deep1M
  • MNIST
  • SIFT100M
  • Text2Image1M

Convert datasets into the .data_new format for compatibility.

Dataset Format

{int: float size in bytes}
{int: number of vectors}
{int: dimension}
{float[]: all vector values, stored sequentially}

Place converted datasets in: 

./dataset/

Evaluation Results

Dataset Recall@10 Query Time Improvement vs. LSH-APG

DAPG achieves higher recall and lower latency on every evaluated dataset.

DAPG vs. LSH-APG

Aspect LSH-APG DAPG Outcome
Query Complexity O(d β(ℓ)) O(d̄DAPG β(ℓ)) Same asymptotic form, smaller constant
Degree Control Fixed Adaptive percentile + global cap Controllable sparsity
Update Cost Rebuild required Incremental amortized update Dynamic maintenance
Proof Coverage Heuristic Formal reachability and connectivity bounds Theoretical support
